npm >= 8.0.0
PostgreSQL >= 13.0

## EXTERNAL SERVICES REQUIRED

### AI Services
- Google Gemini API Key (required)
  Get from: https://makersuite.google.com/
  
- HuggingFace API Token (optional)
  Get from: https://huggingface.co/settings/tokens

### Database
- PostgreSQL 13+ server running
- Database: career_disha_db
- User with full privileges on the database

### Email Service (Optional)
- SMTP server (e.g., Gmail SMTP)
- App-specific password for email account

### Cloud Storage (Optional)
- AWS S3 bucket for file storage
- AWS access key and secret key

### External APIs (Optional for development)
- Coursera API key
- Udemy API key
- EdX API key
- Indeed API key
- GitHub Jobs API key

## ENVIRONMENT VARIABLES REQUIRED

Essential variables (minimum for basic functionality):
- DATABASE_URL
- JWT_SECRET
- JWT_REFRESH_SECRET
- GEMINI_API_KEY

Optional variables (for full functionality):
- SMTP_HOST, SMTP_USER, SMTP_PASS
- AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, S3_BUCKET_NAME
- HUGGINGFACE_API_KEY
- COURSERA_API_KEY, UDEMY_API_KEY, etc.

## SYSTEM RESOURCES

### Minimum System Requirements
- RAM: 2 GB (4 GB recommended)
- Storage: 5 GB free space
- CPU: Dual-core processor
- Network: Internet connection for AI services

### Recommended System Requirements
- RAM: 8 GB or more
- Storage: 20 GB free space (SSD recommended)
- CPU: Quad-core processor or better
- Network: Stable broadband connection

## PORT REQUIREMENTS

Default ports used:
- 3000: Main application server
- 5432: PostgreSQL database
- 5555: Prisma Studio (development only)

Ensure these ports are available or configure alternatives in .env file.

## INSTALLATION COMMAND

To install all dependencies:
```bash
npm install
```

## VERIFICATION COMMANDS

To verify installation:
```bash
node --version    # Should be >= 18.0.0
npm --version     # Should be >= 8.0.0
npm run health    # Should return server health status
```

## NOTES

1. This project uses ES6+ JavaScript features requiring Node.js 18+
2. Database migrations must be run before first startup
3. AI features require valid API keys to function
4. File upload features create local directories automatically
5. Logging system creates log files automatically
6. WebSocket features require socket.io client compatibility
7. All security middleware is preconfigured for production use
8. Rate limiting is enabled by default for all endpoints
9. CORS is configured for local development - update for production
10. Monitoring endpoints provide health checks and metrics

For detailed setup instructions, see SETUP.md
